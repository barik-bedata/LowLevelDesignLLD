# Private Class

অবজেক্ট-ওরিয়েন্টিড প্রোগ্রামিং (OOP) এ **Private Class** হলো এমন একটি ক্লাস যার অ্যাক্সেসিবিলিটি শুধুমাত্র তার প্যারেন্ট বা আউটার (Outer) ক্লাসের ভেতরেই সীমাবদ্ধ থাকে। 

সাধারণত Java বা C# এর মতো প্রোগ্রামিং ল্যাঙ্গুয়েজে সরাসরি কোনো ফাইলের ভেতরে বা টপ-লেভেলে (Top-level) কোনো ক্লাসকে `private` করা যায় না। তবে একটি ক্লাসের ভেতরে অন্য আরেকটি ক্লাস ডিক্লেয়ার করলে (যাকে **Nested Class** বা **Inner Class** বলা হয়), সেটিকে `private` করা সম্ভব।

### কেন ডিক্লেয়ার করা যায় বা যায় না? (Why it can/cannot be declared)

#### ১. টপ-লেভেল ক্লাস (Top-level Class) কেন `private` করা যায় না?
একটি সাধারণ ক্লাস যা সরাসরি ফাইলে বা নেমস্পেস/প্যাকেজের অধীনে থাকে, তাকে টপ-লেভেল ক্লাস বলে। এটিকে `private` করা যায় না, কারণ:
* **ক্লাসটি সম্পূর্ণ অব্যবহারযোগ্য (Useless) হয়ে পড়বে:** প্রোগ্রামিং ল্যাঙ্গুয়েজের নিয়মানুযায়ী, `private` মেম্বার শুধুমাত্র তার নিজস্ব ডিক্লেয়ার্ড স্কোপের (Scope) ভেতর অ্যাক্সেস করা যায়। একটি টপ-লেভেল ক্লাসের প্যারেন্ট স্কোপ হলো তার ফাইল বা প্যাকেজ। যদি কোনো টপ-লেভেল ক্লাসকে `private` করা হতো, তবে সেই ক্লাসের বাইরে অন্য কোনো কোড বা ক্লাস তার অস্তিত্বই জানতে পারতো না। ফলে সেই ক্লাসের কোনো অবজেক্ট তৈরি করা বা তাকে ব্যবহার করা অসম্ভব হয়ে যেত।
* **কম্পাইলার ও আর্কিটেকচারাল ডিজাইন:** অবজেক্ট-ওরিয়েন্টেড ডিজাইনে ক্লাস তৈরি করা হয় অন্য কোডের সাথে ইন্টারঅ্যাক্ট করার জন্য। যেহেতু বাইরের কোনো ক্লাস থেকে এটিকে ব্যবহার করার সুযোগ থাকত না, তাই কম্পাইলার ডিজাইনাররা টপ-লেভেল ক্লাসকে `private` করার অনুমতি দেননি।

*(টপ-লেভেল ক্লাসের জন্য তাই শুধুমাত্র `public` অথবা প্যাকেজ/প্রজেক্ট লেভেলের অ্যাক্সেস মডিফায়ার যেমন Java-তে `default` (package-private) এবং C#-এ `internal` ব্যবহারের অনুমতি দেওয়া হয়)।*

#### ২. নেস্টেড ক্লাস (Nested Class) কেন `private` করা যায়?
যখন কোনো ক্লাস অন্য একটি ক্লাসের ভেতরে ডিক্লেয়ার করা হয়, তাকে নেস্টেড বা ইনার ক্লাস বলে। এটিকে `private` করা যায়, কারণ:
* **আউটার ক্লাসের ভেতরে ব্যবহারের সুযোগ থাকে:** যেহেতু এটি অন্য একটি ক্লাসের ভেতরে রয়েছে, তাই `private` করা হলেও আউটার ক্লাসটি (Outer Class) তার ভেতরের মেথডে এই প্রাইভেট ক্লাসের অবজেক্ট তৈরি করতে পারে এবং এর কাজ ব্যবহার করতে পারে।
* **ইনফরমেশন হাইডিং বা এনক্যাপসুলেশন (Encapsulation):** অনেক সময় এমন ক্লাসের প্রয়োজন হয় যা শুধুমাত্র একটি নির্দিষ্ট ক্লাসের লজিক সম্পন্ন করতে সাহায্য করবে এবং বাইরের কোনো ক্লাসের তার সম্পর্কে জানার প্রয়োজন নেই। যেমন: `LinkedList` ক্লাসের ভেতরে `Node` ক্লাস। `LinkedList` নিজেই `Node` তৈরি ও ব্যবহার করে, কিন্তু বাইরের কোনো কোড থেকে সরাসরি `Node` অবজেক্ট অ্যাক্সেস করতে দেওয়ার প্রয়োজন নেই।

**সংক্ষেপে:**
* **টপ-লেভেল ক্লাস:** `private` করা যায় না, কারণ তাহলে সেটিকে কোথাও থেকেই অ্যাক্সেস বা ব্যবহার করা যাবে না।
* **নেস্টেড ক্লাস:** `private` করা যায়, কারণ বাইরের সবার জন্য বন্ধ থাকলেও আউটার ক্লাসটি নিজে সেটিকে ভেতরে ব্যবহার করতে পারে।

## কিভাবে ডিক্লেয়ার করা হয়? (Syntax & Examples)

### C# Example:
```csharp
public class Car 
{
    // Private Nested Class
    private class Engine 
    {
        public void Start() 
        {
            Console.WriteLine("Engine started...");
        }
    }

    private Engine _engine;

    public Car() 
    {
        _engine = new Engine(); // শুধুমাত্র Car ক্লাসই Engine ক্লাসটি ব্যবহার করতে পারবে
    }

    public void Drive() 
    {
        _engine.Start();
        Console.WriteLine("Car is moving...");
    }
}
```

### Java Example:
```java
public class LinkedList {
    // Private Nested Class
    private class Node {
        int data;
        Node next;

        Node(int data) {
            this.data = data;
            this.next = null;
        }
    }

    private Node head;

    public void add(int value) {
        Node newNode = new Node(value);
        if (head == null) {
            head = newNode;
        } else {
            Node temp = head;
            while (temp.next != null) {
                temp = temp.next;
            }
            temp.next = newNode;
        }
    }
}
```

## Private Class এর সুবিধাসমূহ (Advantages)

1. **উন্নত এনক্যাপসুলেশন (Enhanced Encapsulation):**
   যে ক্লাস বা লজিকটি বাইরের অন্যান্য ক্লাসের জানার প্রয়োজন নেই, সেটিকে সম্পূর্ণরূপে লুকিয়ে ফেলা যায় (Information Hiding)।
2. **কোড মেইনটেইনেবিলিটি (Better Maintainability):**
   যেহেতু ক্লাসটি নির্দিষ্ট একটি ক্লাসের বাইরে ব্যবহার করা যায় না, তাই এই ক্লাসে কোনো পরিবর্তন আনলে বাইরের অন্যান্য কোডে কোনো প্রভাব পড়ে না।
3. **লজিক্যাল গ্রুপিং (Logical Grouping):**
   যদি কোনো ক্লাস শুধুমাত্র অন্য একটি ক্লাসের কাজের সহায়তার জন্য প্রয়োজন হয়, তবে সেটিকে আলাদা ফাইলে না লিখে একই ফাইলের ভেতরে Nested Private Class হিসেবে রাখা পরিষ্কার কোড স্ট্রাকচার তৈরি করে।

## বাস্তবমুখী ব্যবহার (Real-world Use Cases)

* **ডাটা স্ট্রাকচারের নোড (Data Structure Nodes):**
  `LinkedList`, `Tree` বা `Graph` ডিক্লেয়ার করার সময় তাদের ভেতরের `Node` বা `Vertex` ক্লাসকে `private` রাখা হয়, কারণ বাইরের ক্লাসের সরাসরি নোড অ্যাক্সেস করার প্রয়োজন নেই।
* **হেল্পার ক্লাস (Helper / Utility Classes):**
  ধরা যাক, একটি ক্লাসের কাজ সম্পন্ন করার জন্য একটি নির্দিষ্ট ফরম্যাটার বা ডাটা কনভার্টার ক্লাসের প্রয়োজন যা অন্য কোথাও লাগবে না। সেক্ষেত্রে Private Class বেশ উপযোগী।
* **স্টেট প্যাটার্ন বা ইন্টারনাল ওয়ার্কার (Internal Worker / States):**
  কোনো অ্যাপ্লিকেশনের অভ্যন্তরীণ ব্যাকগ্রাউন্ড প্রসেস বা কোনো অবজেক্টের স্টেট পরিবর্তনের লজিক বাইরের অ্যাপ থেকে আড়াল রাখতে এটি ব্যবহার করা হয়।
