# 🗄️ রিয়েল-ওয়ার্ল্ড উদাহরণ: ডেটাবেস কানেকশন ক্লোনিং এবং প্রাইভেট ডেটা পরিবর্তন

প্রোটোটাইপ প্যাটার্নের একটি চমৎকার রিয়েল-ওয়ার্ল্ড ব্যবহার হলো **ডেটাবেস কানেকশন (Database Connection)** তৈরি করা। একটি ডেটাবেস কানেকশন তৈরি করার সময় কিছু ভারী কাজ করতে হয়, যেমন:
* নেটওয়ার্ক পিন (Ping) করা।
* সিকিউরিটি সার্টিফিকেট (TLS/SSL) চেক করা।
* কানেকশন পুল (Connection Pool) তৈরি করা।
* কনফিগারেশন ফাইল (XML/JSON) রিড করা।

এগুলো করতে বেশ সময় লাগে (ধরে নিচ্ছি ১.৫ সেকেন্ড)।

### ❓ সমস্যা (The Problem)
ধরুন, আমাদের অ্যাপ্লিকেশনে ৩ ধরনের ডেটাবেস (MySQL, PostgreSQL, Oracle) কানেক্ট করতে হবে। আমরা যদি ৩ বার `new DatabaseConnection()` কল করি, তবে ৩ বারই উপরের ভারী কাজগুলো হবে (১.৫ x ৩ = ৪.৫ সেকেন্ড নষ্ট হবে)।

আবার, ডেটাবেস কানেকশনের আসল ইউজার আইডি এবং পাসওয়ার্ড থাকে `private` ফিল্ডে (`_connectionString`), যাতে বাহির থেকে কেউ হ্যাক করতে না পারে।

তাহলে আমরা কীভাবে আগের কানেকশনকে ক্লোন করবো, কিন্তু একইসাথে **নতুন ডেটাবেসের প্রাইভেট আইডি-পাসওয়ার্ড** সেখানে বসাবো?

### 💡 সমাধান (The Solution)
আমরা `ICloneable` এর নরমাল `Clone()` মেথডের উপর ভরসা না করে একটি **কাস্টম ক্লোন মেথড** (`CloneWithNewProvider`) তৈরি করবো।

যেহেতু এই মেথডটি ক্লাসের ভেতরেই আছে, সে শ্যালো কপি করে ভারী কনফিগারেশনগুলো দ্রুত কপি করে নেবে, এবং এরপর প্রাইভেট ফিল্ডগুলোতে (যেমন `_connectionString`) নতুন ডেটা বসিয়ে দেবে।

```csharp
public DatabaseConnection CloneWithNewProvider(string newProvider, string newConnectionString)
{
    // ১. শ্যালো কপি (ভারী কনফিগারেশন দ্রুত কপি হয়ে গেলো)
    DatabaseConnection copy = (DatabaseConnection)this.MemberwiseClone();
    
    // ২. প্রাইভেট ডেটা আপডেট (যেহেতু মেথডটি ক্লাসের ভেতরে, তাই প্রাইভেট ফিল্ড অ্যাক্সেস করা যায়)
    copy._dbProvider = newProvider;
    copy._connectionString = newConnectionString;

    return copy;
}
```

### 🚀 লাভ কী হলো? (Benefits)
1. **পারফরম্যান্স:** প্রথমবার MySQL কানেকশন তৈরি করতে ১.৫ সেকেন্ড লেগেছে। কিন্তু যখন আমরা PostgreSQL ও Oracle এর জন্য ক্লোন করলাম, তখন সময় লাগলো একদম শূন্য! কারণ ভারী কাজগুলো আর রিপিট হয়নি।
2. **প্রাইভেট ডেটা মডিফিকেশন:** আমরা খুব সফলভাবে বাহির থেকে (Program.cs থেকে) নতুন কানেকশন স্ট্রিং পাঠিয়েছি এবং সেটা ক্লোন হওয়া অবজেক্টের `private` ফিল্ডে গিয়ে সেট হয়ে গেছে।
