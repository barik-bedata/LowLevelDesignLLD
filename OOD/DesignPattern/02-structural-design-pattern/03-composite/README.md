# Composite Design Pattern (কম্পোজিট প্যাটার্ন)

## 📖 Overview (সংক্ষিপ্ত ধারণা)
Composite Pattern এমন একটি স্ট্রাকচারাল ডিজাইন প্যাটার্ন যা অবজেক্টগুলোকে একটি **"Tree" (গাছ)** এর মতো স্ট্রাকচারে সাজাতে সাহায্য করে। এর সবচেয়ে বড় ম্যাজিক হলো—এটি আপনাকে একটি সিঙ্গেল অবজেক্ট (Leaf) এবং অনেকগুলো অবজেক্টের গ্রুপ (Composite) কে **একইভাবে (Uniformly)** ট্রিট করার সুবিধা দেয়।
### **Composite Design Pattern: Composes objects into tree structures to represent part-whole hierarchies, so clients treat single objects and groups of objects the same way.**

---

## 🛠️ Components (মূল অংশগুলো)
১. **Component:** সবার জন্য একটি কমন ইন্টারফেস।
২. **Leaf:** Tree এর একদম শেষ প্রান্তের অবজেক্ট (যার ভেতরে অন্য কিছু থাকে না)। 
৩. **Composite:** কন্টেইনার অবজেক্ট (যার ভেতরে Leaf বা অন্য Composite থাকতে পারে)।
৪. **Client:** যিনি এই সিস্টেমটি ব্যবহার করেন।

*(বিঃদ্রঃ আমরা এই ফোল্ডারের কোডগুলোতে **Interface Segregation Principle (ISP)** মেনে কোড করেছি। অর্থাৎ `Component` ইন্টারফেসে আমরা `Add/Remove` মেথড না রেখে, শুধু `Composite` গুলোর জন্য আলাদা ইন্টারফেস তৈরি করেছি। এতে কোড ১০০% টাইপ-সেইফ হয়েছে।)*

## 💡 Real-Life Examples (বাস্তব জীবনের উদাহরণ)

এই ফোল্ডারে আমরা কম্পোজিট প্যাটার্নের ৩টি অসাধারণ রিয়েল-লাইফ উদাহরণ কোডসহ দিয়েছি:

### ১. Corporate Hierarchy (কোম্পানির গঠন)
- **কোথায় আছে:** `Composite.cs` ফাইলে।
- **ব্যাখ্যা:** একটি কোম্পানিতে CEO থাকে, তার আন্ডারে Manager থাকে, এবং তাদের আন্ডারে Developer থাকে। HR যদি পুরো কোম্পানির বেতনের হিসেব চায়, তবে সে জাস্ট CEO কে `GetSalary()` কল করবে। CEO তার ম্যানেজারদের জিজ্ঞেস করবে, ম্যানেজাররা তাদের ডেভেলপারদের জিজ্ঞেস করবে, এবং সবশেষে মোট বেতনের হিসেব চলে আসবে। HR কে আলাদা করে সবার কাছে যেতে হবে না।

### ২. File System (কম্পিউটারের ফাইল ও ফোল্ডার)
- **কোথায় আছে:** `FileSystemComposite.cs` ফাইলে।
- **ব্যাখ্যা:** আপনার হার্ডডিস্কের ফোল্ডার স্ট্রাকচার চিন্তা করুন।
  - **Leaf:** `File` (এর নিজস্ব সাইজ আছে, কিন্তু এর ভেতরে কিছু রাখা যায় না)।
  - **Composite:** `Folder` (এর ভেতরে ফাইলও থাকতে পারে, আবার সাব-ফোল্ডারও থাকতে পারে)।
  - আপনি যখন কোনো মেইন ফোল্ডারের `GetSize()` কল করবেন, তখন সেটি নিজে থেকেই তার ভেতরের সমস্ত ফাইল এবং সাব-ফোল্ডারের সাইজ হিসেব করে মোট সাইজটি আপনাকে রিটার্ন করবে।

### ৩. E-Commerce Delivery Box (শপিং বক্স)
- **কোথায় আছে:** `ECommerceDeliveryBox.cs` ফাইলে।
- **ব্যাখ্যা:** দারাজ বা আমাজন থেকে যখন আমরা অনেক কিছু অর্ডার করি, তখন তারা একটা বড় বক্সের ভেতরে ছোট বক্স দেয়, আবার সেই ছোট বক্সের ভেতরে প্রোডাক্ট থাকে।
  - **Leaf:** `SingleItem` (যেমন: মোবাইল বা চার্জার)।
  - **Composite:** `DeliveryBox` (বক্সের ভেতর আইটেম বা আরও ছোট বক্স)।
  - পেমেন্ট করার সময় কাস্টমারকে প্রতিটি আইটেম আলাদা করে গুনতে হয় না। সে শুধু মেইন ডেলিভারি বক্সের `GetPrice()` কল করলেই পুরো প্যাকেজের দাম একবারে ক্যালকুলেট হয়ে যায়!

## 🤔 When to use it? (কখন ব্যবহার করবেন?)
1. যখন আপনার ডেটা একটি **Tree-like (Hierarchical) Structure** ফলো করে।
2. যখন আপনি চান ক্লায়েন্ট কোড যেন সিঙ্গেল অবজেক্ট এবং অবজেক্টের গ্রুপ—দুটোকেই কোনো পার্থক্য ছাড়াই একই নিয়মে ব্যবহার করতে পারে।
