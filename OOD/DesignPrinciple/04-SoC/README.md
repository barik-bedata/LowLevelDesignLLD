# SoC (Separation of Concerns)

**মূল কথা:** প্রতিটি অংশের কাজ আলাদা রাখুন।

**বিস্তারিত ব্যাখ্যা:**
একটি প্রোগ্রামের বিভিন্ন কাজ বা দায়িত্বকে আলাদা আলাদা অংশে ভাগ করাকে Separation of Concerns বলে। যেমন: ডাটাবেস থেকে ডেটা আনার কাজ, ইউজার ইন্টারফেসে ডেটা দেখানোর কাজ এবং বিজনেস লজিক—এগুলো সব আলাদা আলাদা ক্লাস বা মডিউলে থাকা উচিত। এতে একটি অংশে পরিবর্তন আনলে অন্য অংশগুলোর ওপর কোনো প্রভাব পড়ে না।

**C# উদাহরণ:**
এই ফোল্ডারের `SoC.cs` ফাইলে কীভাবে আলাদা আলাদা ক্লাসে ভিন্ন ভিন্ন দায়িত্ব ভাগ করে দেওয়া হয়েছে, তার C# কোড উদাহরণ দেওয়া আছে।
