# DRY (Don't Repeat Yourself)

**মূল কথা:** একই কোড বারবার লিখবেন না।

**বিস্তারিত ব্যাখ্যা:** 
DRY প্রিন্সিপালের মূল লক্ষ্য হলো কোডের ডুপ্লিকেশন কমানো। যদি আপনার প্রজেক্টের একাধিক জায়গায় একই লজিক বা কোড ব্লক ব্যবহার করা হয়, তবে কোনো পরিবর্তনের দরকার হলে আপনাকে সব জায়গায় পরিবর্তন করতে হবে, যা ভুল হওয়ার সম্ভাবনা বাড়িয়ে দেয়। তাই কমন কোডগুলোকে একটি আলাদা ফাংশন, মেথড বা ক্লাসে নিয়ে আসতে হয়। এতে কোড মেইনটেইন করা সহজ হয় এবং একবার পরিবর্তন করলেই সব জায়গায় আপডেট হয়ে যায়।

**C# উদাহরণ:**
এই ফোল্ডারের `DRY.cs` ফাইলে DRY প্রিন্সিপাল অনুসরণ না করার সমস্যা এবং কীভাবে তা সমাধান করতে হয় তার C# কোড উদাহরণ দেওয়া আছে।
