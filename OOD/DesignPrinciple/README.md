# Design Principles (ডিজাইন প্রিন্সিপালস)

সফটওয়্যার ইঞ্জিনিয়ারিংয়ে **Design Principles** হলো কিছু সাধারণ নিয়ম বা গাইডলাইন, যা মেনে কোড লিখলে কোড অনেক বেশি ক্লিন, মেইনটেইনেবল এবং বাগ-ফ্রি হয়। এগুলো কোনো নির্দিষ্ট ল্যাঙ্গুয়েজের জন্য নয়, বরং সব ল্যাঙ্গুয়েজেই ব্যবহার করা যায়।

যেহেতু আপনার `SOLID` এর জন্য আলাদা ফোল্ডার আছে, তাই এখানে আমরা সফটওয়্যার ডেভেলপমেন্টের অন্য ৪টি সবচেয়ে গুরুত্বপূর্ণ এবং বেসিক প্রিন্সিপাল নিয়ে আলোচনা করবো। প্রতিটি প্রিন্সিপালের জন্য আলাদা ফোল্ডার করে সেখানে বিস্তারিত ব্যাখ্যা এবং C# এর উদাহরণ দেওয়া হয়েছে:

### [১. DRY (Don't Repeat Yourself)](./01-DRY)
একই কোড বারবার লিখবেন না। বিস্তারিত জানতে এবং কোড উদাহরণ দেখতে [এখানে ক্লিক করুন](./01-DRY/README.md)।

### [২. KISS (Keep It Simple, Stupid)](./02-KISS)
কোড যতটা সম্ভব সহজ রাখুন। বিস্তারিত জানতে এবং কোড উদাহরণ দেখতে [এখানে ক্লিক করুন](./02-KISS/README.md)।

### [৩. YAGNI (You Aren't Gonna Need It)](./03-YAGNI)
ভবিষ্যতে লাগতে পারে ভেবে আজই কোনো কোড লিখবেন না। বিস্তারিত জানতে এবং কোড উদাহরণ দেখতে [এখানে ক্লিক করুন](./03-YAGNI/README.md)।

### [৪. SoC (Separation of Concerns)](./04-SoC)
প্রতিটি অংশের কাজ আলাদা রাখুন। বিস্তারিত জানতে এবং কোড উদাহরণ দেখতে [এখানে ক্লিক করুন](./04-SoC/README.md)।
